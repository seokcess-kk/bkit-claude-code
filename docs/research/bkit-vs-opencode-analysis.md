## 2. 핵심 철학 비교

### bkit - "AI-Native Development Methodology"
```
핵심 철학:
├── Automation First: 명시하지 않아도 자동 작동
├── No Guessing: 모르면 문서 확인 → 없으면 질문
├── Docs = Code: 설계 문서가 코드의 근원지
└── PDCA Cycle: Plan-Do-Check-Act 반복
```

### oh-my-opencode - "Batteries-Included Agent Harness"
```
핵심 철학:
├── 전문화된 에이전트 협업: Sisyphus + Oracle + Librarian + Explore
├── 병렬 작업 처리: 여러 에이전트 동시 실행
├── ultrawork 모드: 고급 기능 일괄 활성화
└── Claude Code 호환성: 크로스 플랫폼 지향
```

### opencode - "Open Source AI Coding Agent"
```
핵심 철학:
├── 100% 오픈소스: MIT 라이선스
├── 모델 중립성: Claude, OpenAI, Google 등 다양한 모델
├── 제공자 비종속: 특정 AI 회사에 종속되지 않음
└── 터미널 우선: Neovim 사용자 친화적 UI
```

## 6. 강점 및 약점 분석

### 6.1 bkit

**강점:**
- ✅ **체계적인 방법론**: PDCA + 9단계 파이프라인으로 일관된 개발 프로세스
- ✅ **자동화 극대화**: Hook 시스템으로 수동 작업 최소화
- ✅ **설계-구현 동기화**: Gap Analysis로 문서와 코드 일치도 자동 검증
- ✅ **프로젝트 레벨 최적화**: Starter/Dynamic/Enterprise 맞춤 가이드
- ✅ **풍부한 템플릿**: 21개 템플릿으로 문서화 표준화
- ✅ **Zero Script QA**: 테스트 코드 없이 로그 기반 검증

**약점:**
- ❌ **Claude 종속**: Claude Code 전용, 다른 AI 모델 사용 불가
- ❌ **복잡한 학습 곡선**: 20개 커맨드, 11개 에이전트, 18개 스킬 이해 필요
- ❌ **Bash 의존**: 스크립트가 Bash로 작성되어 Windows 호환성 이슈

### 6.2 oh-my-opencode

**강점:**
- ✅ **병렬 에이전트**: 여러 전문 에이전트 동시 실행으로 효율성 극대화
- ✅ **내장 MCP**: Exa, Context7, Grep.app으로 외부 정보 즉시 접근
- ✅ **LSP/AST 도구**: 정교한 리팩토링 지원
- ✅ **ultrawork 모드**: 단일 키워드로 고급 기능 일괄 활성화
- ✅ **Claude Code 호환**: 크로스 플랫폼 이식성

**약점:**
- ❌ **OpenCode 종속**: OpenCode 플랫폼에서만 동작
- ❌ **방법론 부재**: PDCA 같은 체계적 개발 프로세스 없음
- ❌ **문서화 미흡**: 템플릿/문서 표준화 기능 없음

### 6.3 opencode

**강점:**
- ✅ **완전 오픈소스**: MIT 라이선스로 자유로운 사용/수정
- ✅ **모델 중립**: Claude, OpenAI, Google 등 다양한 AI 선택
- ✅ **독립 실행**: 플러그인이 아닌 독립 앱으로 즉시 사용
- ✅ **활발한 커뮤니티**: 82.7k 스타, 675+ 릴리스
- ✅ **데스크톱 앱**: macOS/Windows/Linux 모두 지원
- ✅ **터미널 UI 최적화**: Neovim 사용자 친화적

**약점:**
- ❌ **방법론 부재**: 개발 프로세스 가이드 없음
- ❌ **제한된 에이전트**: 2개 에이전트 (Build, Plan)만 제공
- ❌ **플러그인 생태계 미성숙**: oh-my-opencode 같은 확장 의존

1. **병렬 에이전트 강화**: oh-my-opencode의 병렬 실행 패턴 참고 가능

3. **모델 중립성**: opencode의 다중 모델 지원은 장기적 고려 사항
4. **방법론 우위 유지**: PDCA + 파이프라인은 bkit만의 강력한 차별점
